# SPEC 11 — RELEASE & CHANGE MANAGEMENT SPEC (FROZEN)

## Purpose

Prevent silent drift as the app evolves.

---

## Change Rules

Any change affecting:
- voice
- intake structure
- time pacing
- memory behavior

requires review against Specs 1–5

---

## Versioning

- Intake changes = version bump
- Voice changes = version bump
- Timeframe changes = version bump

---

## Communication Rule

If a change affects user experience:
- explain it calmly
- no hype
- no urgency
