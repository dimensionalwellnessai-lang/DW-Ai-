Change #3: Replace regex “sync extraction” with STRUCTURED actions from the AI

This is the biggest “flow catch” fix.

Right now you extract items from chat using regex in:
	•	server/routes.ts → extractSyncableItems(userMessage, aiResponse)

That will always be hit-or-miss.

Instead: make the AI return a structured payload like:

{
  "response": "…normal assistant text…",
  "actions": [
    {
      "type": "schedule_block",
      "title": "Study block",
      "startTime": "2026-01-08T18:00:00-08:00",
      "endTime": "2026-01-08T19:30:00-08:00",
      "dimensionTags": ["school", "mind"]
    }
  ]
}

Where:
	•	Server: update /api/chat/smart in server/routes.ts
	•	stop relying on regex
	•	validate actions with zod (you already use zod elsewhere)
	•	Client: in client/src/components/ai-workspace.tsx
	•	when chat returns actions, immediately feed them into the Sync Tray (or even auto-approve if user chooses that mode)

Result: “I told the AI my situation → it created structure → I confirm → it’s in my life.” That’s your product.
